# Differentiators

**Status:** Defined  
**Stage:** 0

---

## Core Differentiator

Stareha's differentiator is not chat.

It is:
```
Workflow memory + proactive preparation + transparent learning
```

---

## Compared to Existing Tools

### ChatGPT / Claude
Great for conversation. Does not understand local work context across terminal, browser, files, and learning sessions.

### Cursor / Claude Code
Great for coding tasks. Mostly editor or prompt-based. Does not deeply remember the user's broader workflow across sessions.

### Browser History
Stores URLs. Does not understand learning progress, decisions, or project context.

### Terminal History
Stores commands. Does not understand patterns, mistakes, what worked, or next steps.

### Courses
Generic learning paths. Not personalized to actual user behavior and current skill gaps.

### Notion / Obsidian
Great for manual notes. Requires the user to actively document. Stareha learns automatically.

---

## What Stareha Does Differently

**It learns automatically from approved sources:**
- Terminal commands
- Claude Code conversations
- Browser learning/research
- App usage patterns
- Files and project activity
- Errors and debugging attempts
- User goals and preferences

**It converts activity into:**
- Workflow memory
- Learning profile
- Personalized quizzes
- Exercises
- Suggestions
- Session summaries
- Next-step guidance

**It becomes more useful over time** — not just better at answering, but better at knowing what the user needs before they ask.

---

## The Key Insight

Most AI tools are reactive.
- User asks → AI answers.

Stareha is proactive.
- Stareha observes → prepares → suggests.

The companion loop:
```
Observe → Learn → Prepare → Guide → Observe again
```

This loop is what no current tool provides end-to-end.
